Getting Started

Set up Context Chain step by step

Checking status... 0%
Quick Start Complete steps 1-5, then go to Quick Scan — select a repo and see design decisions automatically. No goal or business context needed.
1
Install Prerequisites
You need Claude CLI to power the AI analysis. Install it and log in:
bashnpm install -g @anthropic-ai/claude-code claude login
Or switch to Anthropic API in the Quick Scan page and enter your API key instead — no CLI needed.
Checking...
Tip Run bash scripts/setup.sh to check all prerequisites and pull Docker images.
2
Configure Repositories
Add your code repositories to ckg.config.json. Each repo needs a name, absolute path, type, and CPG file path.
Checking config...
json{ "project": "my-project", "ai": { "provider": "claude-cli" }, "repos": [ { "name": "my-service", "path": "/absolute/path/to/repo", "type": "backend", "cpgFile": "data/my-service.json", "packages": [] } ] }
Go to System page to add repos
3
Generate Code Property Graphs
Use Joern to scan each repository and produce a CPG JSON file. This extracts functions, files, and call relationships from your source code.
Checking CPG files...
bash# From the context-chain directory, for each repo: joern --script joern/extract-code-entities.sc \ --param cpgFile=my-service.cpg.bin \ --param outFile=data/my-service.json \ --param repoName=my-service
Note Run Joern from within each repo directory first to generate the .cpg.bin file, then use the extraction script to produce the JSON.
4
Start Memgraph
Start the graph database using Docker Compose. Memgraph stores your code entities and design decisions as a connected graph.
Checking Memgraph...
bashdocker compose up -d # Memgraph: localhost:7687 # Memgraph Lab: localhost:3000
5
Initialize Database
Run Full Setup to create the schema, ingest all CPG files into Memgraph, and link cross-repo dependencies. You can also do this from the System page.
Checking graph data...
bash# All-in-one: npm run db:schema && npm run ingest:all && npm run link:repos # Or use the System page's "Full Setup" button
Go to System page
6
Extract Design Decisions
Use Scan to point at any code directory. The AI automatically identifies the most important files, reads your source code, and extracts design decisions, tradeoffs, and patterns. No configuration needed.
Checking decisions...
Open Scan
7
Connect to Claude Code via MCP
Start the MCP server so Claude Code can query your design decisions while coding. Add the server config to your project's .mcp.json.
bash# Start MCP server: npm run mcp
json// Add to your project's .mcp.json: { "mcpServers": { "context-chain": { "command": "bash", "args": ["/path/to/context-chain/mcp-start.sh"] } } }
Tip Once connected, Claude Code can automatically look up relevant design decisions when you ask it to modify code. It provides 9 tools including get_context_for_code, search_decisions_by_keyword, and get_decision_relationships.
